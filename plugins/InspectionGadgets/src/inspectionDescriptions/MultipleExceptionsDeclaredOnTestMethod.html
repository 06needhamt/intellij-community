Reports JUnit test method throws clauses with more than one exception. Such clauses are unnecessarily verbose. The test methods will not be called from other project code, so there is not need to handle the exceptions separately.

For example:


  public class EssentialTest {

    @Test
    public void testUnknownLetters() throws FileNotFoundException, UnsupportedEncodingException {
      HunspellDictionary hunspell = getTestHunspell();
      assertNull(hunspell.contains("русский"));
    }

    // ...
  }

After the quick fix is applied, the result looks like this:


  public class EssentialTest {

    @Test
    public void testUnknownLetters() throws Exception {
      HunspellDictionary hunspell = getTestHunspell();
      assertNull(hunspell.contains("русский"));
    }

    // ...
  }
