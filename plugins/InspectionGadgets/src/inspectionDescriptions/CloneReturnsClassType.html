Reports clone() methods that have a return type different than the class they are contained in. Often a clone() method will have a return type of java.lang.Object, which makes it harder to use by its clients. Effective Java (second and third edition) recommends for the return type of the clone() method to be the class type of the object it returns.

Example: 


  class Foo implements Cloneable {
    public Object clone() {
      try {
        super.clone();
      } catch (CloneNotSupportedException e) {
        throw new AssertionError();
      }
    }
  }
