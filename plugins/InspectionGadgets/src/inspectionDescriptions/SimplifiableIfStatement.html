Reports if statements that can be replaced with condition using &&, ||, ==, != or ?: operator. The result is usually shorter, but not always more clear, so it's not advised to apply it everywhere.

Example:

  if (condition) return true; else return foo;

After the fix is applied, the result will be like this:

  return condition || foo;

Use the first checkbox below to skip warning when ?: operator is suggested. In this case only &&, ||, == and != suggestions will be highlighted. The fix action to replace with ?:will still be available in the editor.

Use the second checkbox below to skip warning on if-else chains. The fix action will still be available in the editor.
