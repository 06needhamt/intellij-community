Reports protected members in finalclasses.

It is better to declare such members private or package-visible instead.

Reports protected members in the finalclasses. As the final classes are closed to inheritance marking the method as protected is confusing. It's better to declare such members as private or package-visible instead.

Example:

record Bar(int a, int b) {
  protected sum() { 
     return a + b;
  }
}

After the quick-fix is applied:

record Bar(int a, int b) {
  sum() { 
     return a + b;
  }
}
As shown in the example, class may be explicitly or implicitly final e.g., enum or record.
