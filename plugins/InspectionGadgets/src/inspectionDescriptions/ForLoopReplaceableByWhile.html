Reports for loops which contain neither initialization or update components, and suggests rewriting it as a while loop. This makes the code easier to read.

Example:

  for(; exitCondition(); ) {
    process();
  }

After the quick-fix is applied the result looks like:

  while(exitCondition()) {
    process();
  }
A quick-fix action is also available for other for loops, so you can replace any for loop with a while loop.

Use the checkbox below if you wish this inspection to ignore for loops with trivial or non-existent conditions.
