Reports known Javac issues, performance problems and incompatibilities, e.g. type inference may be slow when it has to process many nested calls: 
  Arrays.asList(                  //warning: Vararg method call with 50+ poly arguments
      Arrays.asList("a", "b"),
      ...
      Arrays.asList("a100", "b100"));
The fix adds explicit type arguments which makes compilation and IDE processing much faster: 
  //noinspection RedundantTypeArguments (explicit type arguments speedup compilation and analysis time)
  Arrays.<List<String>>asList(
      Arrays.asList("a", "b"),
      ...
      Arrays.asList("a100", "b100"));
