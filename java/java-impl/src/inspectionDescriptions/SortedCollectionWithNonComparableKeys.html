Reports construction of sorted collections like new TreeSet<>() which rely on natural order, but their element type does not implement the Comparable interface. Such collection will unlikely to work properly.

A false-positive is possible if collection element is a super-type which is not comparable, but it's intended to hold only comparable sub-types. This is however error-prone. It's better to narrow the collection element type or declare the super-type as comparable.

An option is provided to suppress the warnings on type parameters.

New in 2018.3
