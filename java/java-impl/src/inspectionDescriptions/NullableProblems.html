Reports problems related to nullability annotations.

Examples:

  • Not annotated overridden methods: 
          abstract class A {
          @NotNull abstract String m();
      }
      class B extends A {
          String m() { return "empty string"; }
      }
  • Annotated primitive types: @NotNull int myFoo;
  • Both @Nullable and @NotNull are present for the same member: @Nullable @NotNull String myFooString;
  • Collection of nullable elements is assigned into a collection of non-null elements: 
          void testList(List<@Nullable String> nullableList) {
         List<@NotNull String> list2 = nullableList;
      }

Use the button below to configure nullability annotations and the checkboxes to fine-tune where the inspection should provide warnings.

This inspection only reports if the language level of the project or module is 5 or higher, and if nullability annotations are available in the project
