Reports methods and constructors where constant charset String literal is used e.g. "UTF-8" which can be replaced with a predefined StandardCharsets.UTF_8 code. This may work faster, because charset lookup becomes unnecessary. Also catching UnsupportedEncodingException may become unnecessary as well. In this case the catch block will be removed automatically.

Example:

  try {
    byte[] bytes = "str".getBytes("UTF-8");
  } catch (UnsupportedEncodingException e) {
  }

After quick-fix is applied, the result will be:

  byte[] bytes = "str".getBytes(StandardCharsets.UTF_8);

This inspection is available since Java 7 only.
