Reports classes, interfaces and methods that extend, implement, or override API elements marked with @ApiStatus.NonExtendable.

The @ApiStatus.NonExtendable annotation indicates that the class, interface, or method must not be extended, implemented, or overridden. It is a common case when such interfaces are cast to the library internal implementation, thus if a client provides a different implementation of a class or interface marked with this annotation user will get ClassCastException. Adding new abstract methods to such classes and interfaces will break the compatibility with the client's implementations.
